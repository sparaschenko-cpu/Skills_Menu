# ddxcmd-coverage-check — контроль покрытия Manual/Apps по артефактам кейса

## Назначение

Используй в конце (и/или после больших итераций), чтобы заполнить `15_coverage_check.md`: какие секции Manual/Apps реально применены и где есть пробелы.

## Параметры

- `case_id=YYYYMMDD_slug` (обязательно)

## Входы

- `Artifacts/DDx/<case_id>/15_coverage_check.md`
- `.cursor/skills/ddx-run-analysis/coverage-checklist.md`
- Папка кейса `Artifacts/DDx/<case_id>/` (фактические артефакты)

## Шаги

1. Открой `coverage-checklist.md` и выпиши секции Manual/Apps, которые должны быть покрыты в этом кейсе (по протоколу/триггерам).
2. Сверь с фактическими файлами/содержимым кейса:
   - какие артефакты реально созданы/заполнены
   - где есть пустые шаблоны
3. Заполни `15_coverage_check.md`:
   - “Covered”: секция → где отражена (файл + краткая ссылка/маркер)
   - “Not covered”: секция → причина (не триггерилось / пропущено / отложено)
   - “Next actions”: 3–7 конкретных шагов для закрытия пробелов

## Выходы (Artifacts/DDx/<case_id>/...)

- `15_coverage_check.md` (обновлён)

## Quality gates

- Не заявляй “covered” без указания конкретного файла и места/маркерной строки в нём.
