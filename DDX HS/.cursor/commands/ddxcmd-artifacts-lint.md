# ddxcmd-artifacts-lint — консистентность артефактов кейса (structure + chunking)

## Назначение

Используй, чтобы быстро проверить структуру `Artifacts/DDx/<case_id>/`: индексы, чанки, диапазоны A/H, отсутствие “дыр” в ID и соответствие правилу `case_id`.

## Параметры

- `case_id=YYYYMMDD_slug` (обязательно)

## Входы

- `Artifacts/DDx/<case_id>/`
- Скелет: `Artifacts/DDx/_TEMPLATES/case_skeleton/` (как эталон состава)

## Шаги

1. Проверь формат `case_id`.
2. Проверь обязательные файлы минимального пакета (см. скелет).
3. Проверь chunking:
   - `02_annotations/index.md` существует и ссылается на чанки `A-....md`
   - `06_hcards/index.md` существует и ссылается на чанки `H-....md`
4. Проверь “дыры”:
   - если встречаются A-IDs/H-IDs, проверь, что диапазоны файлов покрывают их без пропусков (или что пропуски объяснены).
5. Верни список проблем (если есть) и remediation-план (какой файл создать/переименовать/обновить индекс).

## Выходы

- Отчёт в чате: PASS или список проблем + шаги исправления.

## Quality gates

- Не модифицируй файлы автоматически без явного запроса пользователя.
