# ddxcmd-case-status — статус кейса и готовность артефактов

## Назначение

Используй, чтобы быстро понять: какие артефакты кейса уже заполнены, где пусто, какие индексы/чанки ожидаются, и к какой фазе кейс готов — без запуска полного анализа.

## Параметры

- `case_id=YYYYMMDD_slug` (обязательно)

## Входы

- Папка кейса: `Artifacts/DDx/<case_id>/`

## Шаги

1. Проверь наличие ключевых файлов (минимальный пакет A/B/C):
   - `00_question_and_scope.md`, `01_zero_point.md`, `03_data_map.md`, `04_lenses.md`, `05_smh_registry.md`, `07_evidence_matrix.md`, `08_hypothesis_map.md`, `14_conclusion_3_3_3.md`, `15_coverage_check.md`
2. Проверь chunking-дисциплину:
   - есть ли `02_annotations/index.md` и хотя бы один чанк `02_annotations/A-....md`
   - есть ли `06_hcards/index.md` и хотя бы один чанк `06_hcards/H-....md`
3. Быстро классифицируй готовность:
   - **готов к Phase 1**: есть `00_question_and_scope.md` и структура аннотаций
   - **готов к Phase 2**: есть Data Map + первичный набор аннотаций/кодов
   - **готов к Phase 3**: есть SMH + Evidence Matrix + Hypothesis Map
4. Выведи список “следующих 3 шагов” (что именно заполнить/какую команду запустить).

## Выходы

- Сообщение в чате: чек-лист состояния + рекомендации по следующим шагам.

## Quality gates

- Не выдумывай наличие файлов: опирайся на фактическое содержимое папки кейса.
