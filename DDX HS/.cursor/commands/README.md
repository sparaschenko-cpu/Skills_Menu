# DDX Commands (`ddxcmd-*`)

Project-level Commands for Cursor (см. `https://cursor.com/docs/context/commands`). Команды доступны через `/` в чате.

## Нейминг

- Каждая команда — это файл `.md` в этой папке.
- Имя команды = имя файла без расширения.
  - пример: файл `ddxcmd-new-case.md` → команда `/ddxcmd-new-case`
- Неймспейс: **`ddxcmd-*`** (чтобы не конфликтовать с Skills `/ddx-...` и Subagents `/ddx-...`).

## Язык

- RU-primary, EN в скобках для ключевых терминов.

## Семантика (важно)

- Commands — это **markdown-промпты**, они не исполняются как код.
- Команда должна **явно инструктировать** агента:
  - какие Skills вызвать через `/ddx-...` (в т.ч. skills с `disable-model-invocation: true`)
  - каких Subagents вызвать через `/ddx-...` (см. `.cursor/rules/ddx-workspace-context.mdc`)
  - какие артефакты создать/обновить в `Artifacts/DDx/<case_id>/...`

## Параметры (единый формат)

Команда должна понимать параметры, переданные пользователем после имени команды:

- `case_id=YYYYMMDD_slug` (обязателен для команд, которые пишут/читают артефакты)
- `input=` (что анализируем / где входной материал)
- `protocol=basic|standard|max`
- `scope=` (что включить/исключить)

Если `case_id` отсутствует — команда должна **сначала запросить его** и не начинать файловые операции.

## Шаблон команды (обязательная структура файла)

Каждый `ddxcmd-*.md` должен содержать:

- **Назначение**
- **Параметры**
- **Входы**
- **Шаги**
- **Выходы (Artifacts/DDx/<case_id>/...)**
- **Quality gates** (A-ID, Axis5, ссылки `Manual/App`, chunk/index дисциплина)
