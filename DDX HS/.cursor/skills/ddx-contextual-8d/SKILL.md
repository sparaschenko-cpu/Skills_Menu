# 8D-режим: контекстуализация анализа (Contextual 8D)

## Критерии активации

Активировать 8D если выполняется хотя бы один:

- [ ] Текст по природе **реляционный** (транскрипт сессии, переписка, диалог, семейная/рабочая коммуникация).
- [ ] Требуется различить **диспозицию** (устойчивая черта) **vs ситуацию** (реакция на контекст).
- [ ] Высок риск **фундаментальной ошибки атрибуции** (чрезмерное приписывание причин внутренним факторам).

Активация фиксируется как мета-решение (Интенция 4.1).

---

## Модифицированная формула 8D

6D: `[Axis1] -> [Axis2] ([Axis3 @ Axis4]) => [Claim] {Axis5, Axis6}`

8D: `[Axis1] -> [Axis2] ([Axis3 @ Axis4 within Axis7 & Axis8]) => [Claim] {Axis5, Axis6}`

Смысл: действие над объектом рассматривается **внутри** ситуационного и реляционного контекстов.

---

## Workflow кодирования Ось 7: Ситуационный контекст (Scene)

Формат: `7.1: <...> / 7.2: <...> / 7.3: <...>`

| Уровень | Что описывает | Что заполнять |
|---------|-------------|---------------|
| **7.1 Микро** | "Здесь и сейчас" | Место, время, непосредственная деятельность |
| **7.2 Мезо** | Жизненная сфера | Работа / Семья / Здоровье / Досуг / Интимные отношения |
| **7.3 Макро** | Социокультурная среда | Культурные нормы, экономические условия, значимые общественные события |

**Пример:** `7.1: Кабинет начальника, вечер после сдачи проекта / 7.2: Профессиональная сфера / 7.3: Корпоративная культура "достигаторства" в IT`

## Workflow кодирования Ось 8: Реляционный контекст (Actors)

Формат: `8.1: <...> / 8.2: <...> / 8.3: <...> / 8.4: <...>`

| Уровень | Что описывает | Что заполнять |
|---------|-------------|---------------|
| **8.1 Интраперсональный** | Внутренний диалог | С кем ведётся: Внутренний критик / Внутренний ребёнок / Конфликт субличностей |
| **8.2 Интерперсональный** | Реальное взаимодействие | Диада / Триада / Малая группа (кто именно) |
| **8.3 Системный** | Крупные системы | Семья / Организация / Социальная сеть (даже без присутствия всех) |
| **8.4 Трансперсональный** | Символические фигуры | "Обобщённый другой" / Бог / Умершие / Культурные герои |

**Пример:** `8.1: Диалог с интроектом критикующего отца / 8.2: Диада субъект-начальник / 8.3: Н/П / 8.4: Н/П`

**Правило:** для ключевых выводов определять контекст минимум на **2 уровнях** каждой оси.

## Новые атрибуты гипотез в 8D

При работе в 8D к H-card (Skill `ddx-hypothesis-lifecycle`) добавляются 2 атрибута:

### Контекстуальная зависимость (Ось 7)

- **Специфичная:** феномен только в узком классе ситуаций (напр. тревога только на работе).
- **Генерализованная:** феномен в широком спектре ситуаций.

### Реляционная функция (Ось 8)

- **Интрапсихическая:** регуляция внутреннего состояния (снижение тревоги, поддержание самооценки).
- **Интерперсональная:** регуляция дистанции / воздействие на другого (привлечение внимания, избегание близости).

### Процедурное правило

**Генерализованная** гипотеза не может стать **центральной** без прохождения Сценария C3 (см. ниже).

## 3 контекстуальных сценария

### C1: Анализ контекст-триггеров (Context Triggers)

**Цель:** выявить, какие ситуации (Ось 7) систематически "запускают" ключевые паттерны.

**Шаги:**
1. Выбрать центральный паттерн (напр. самокритика).
2. Отобрать все аннотации, где паттерн проявляется.
3. Для каждой -- зафиксировать Ось 7 (все 3 уровня).
4. Сгруппировать: в каких контекстах паттерн максимален? В каких отсутствует?
5. Сформулировать гипотезу о триггерах.

### C2: Реконструкция интерперсонального поля (Interpersonal Field)

**Цель:** смоделировать систему значимых отношений и повторяющиеся транзакции.

**Шаги:**
1. Каталогизировать всех "действующих лиц" (с уровнями Оси 8).
2. Для каждой фигуры -- собрать атрибуции и роли, которые даёт субъект.
3. Реконструировать типичные транзакции: стимул -> внутренняя реакция -> внешняя реакция -> ответ.
4. Построить карту интерперсонального поля (социограмму).
5. Квалифицировать ключевые отношения через Ось 6.

### C3: Диспозиция vs ситуация (Disposition vs Situation)

**Цель:** проверить, является ли паттерн устойчивой чертой или реакцией на контекст.

**Шаги:**
1. Сформулировать H-Disp (генерализованная черта) и H-Sit (ситуационная реакция).
2. Создать 2 выборки аннотаций: Контекст 1 (напр. работа) и Контекст 2 (напр. дом).
3. Сравнить: частота? интенсивность? форма? функция?
4. **Вывод:**
   - Консистентно в обоих -> аргумент за H-Disp.
   - Варьирует -> аргумент за H-Sit.
   - Смешанный результат -> оптимизировать: "черта + ситуационная модуляция".

Подробности: `ddx-contextual-playbooks.mdc`.

Связанные правила: `ddx-8d-context.mdc`, `ddx-contextual-playbooks.mdc`, `ddx-annotation.mdc`, `ddx-safety.mdc`.
