# Чек-листы протоколов и артефакты

## Протокол A: Базовый (Basic)

### Обязательные минимумы

- [ ] Диагностический вопрос сформулирован
- [ ] Сокращённый Zero Point выполнен
- [ ] Аннотации: ID + источник + цитата + Ось 3 + Ось 5 + вывод + мета-рефлексия (Блок 4)
- [ ] Минимум 1 цикл H1 vs H0 (Fast Falsification)
- [ ] Каждая H -- с фальсифицируемыми индикаторами
- [ ] Заключение: принятая H + отвергнутая H + Ось 5 + рефлексия

### Выход

1-2 рабочие гипотезы + их индикаторы + краткое обоснование.

## Протокол B: Стандартный (Standard)

### Phase 0

- [ ] Диагностический вопрос + Ось 3 + Ось 4 + политика риска
- [ ] Полный Zero Point (Skill `ddx-safety-audit`)
- [ ] Экспликация теоретических преференций
- [ ] Решение о 8D-режиме

### Phase 1

- [ ] Полное открытое кодирование (все 6 осей)
- [ ] Карта данных построена
- [ ] Выбор первичной оптики: применён Lens Selection Framework (Прил.6) → выбраны 2-3 линзы (Primary + Counter + при необходимости 3-я) с обоснованием
- [ ] Все критерии перехода выполнены

### Phase 2

- [ ] СМГ-реестр создан
- [ ] Генерация через все 3 источника + Ось 2
- [ ] Каждая H в H-card с полной атрибуцией
- [ ] Fast Falsification для каждой значимой пары
- [ ] Байесовские логи ведутся
- [ ] H-Adaptive для каждой H-Pathology
- [ ] Если пул > 7: Воронка Валидации
- [ ] Критерии перехода выполнены

### Phase 3

- [ ] Структурная карта (иерархия + сеть)
- [ ] Динамическая квалификация (Ось 6)
- [ ] Углублённые сценарии (по необходимости)
- [ ] Pre-conclusion Red Teaming
- [ ] Заключение по шаблону 3.3.3

### Встроенные safety-точки

- [ ] После Axis5 4-5 -> низкорисковая проверка
- [ ] При пато-уклоне -> Mode 2

---

## Протокол C: Максимальный (Max)

Всё из Standard, плюс:

- [ ] Формальная активация режимов (Прил.2, 0.0.3)
- [ ] Mode 1 (состязательная генерация) + Воронка
- [ ] Mode 2 (ресурсный анализ) + Ресурсная карта -> разделы 5.3-5.4
- [ ] Mode 3 (мета-аудит) + Паспорт надежности -> раздел 6.2
- [ ] Выбор линз (Прил.6) выполнен и задокументирован (Primary/Counter) перед Фазой 2
- [ ] Все углублённые сценарии 2.2-2.3 (обязательно)
- [ ] 8D-режим (если текст хоть отчасти реляционный)
- [ ] Внешняя супервизия (если ставки высоки)
- [ ] Рабочий лист Воронки Валидации заполнен
