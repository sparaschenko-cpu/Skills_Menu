# Жизненный цикл гипотез (Hypothesis Lifecycle)

## 1. Workflow "Генерация" (Generation)

Для исчерпывающего поля гипотез используй **все три источника**, а не только данные.

### Источник 1: от Данных (Bottom-Up) -- приоритетный

Карта данных (Фаза 1) -> "горячие точки" (насыщенные категории, паттерны, противоречия) -> гипотеза.

### Источник 2: от Модели (Top-Down)

Теоретическая линза как чек-лист -> "слепые зоны" (предсказываемые моделью, но пропущенные феномены) -> гипотеза.

**Правило (Прил.7, 2.2 + Прил.6):** перед Top-Down генерацией в Standard/Max выбрать 2-3 комплементарные линзы (**Primary + Counter + при необходимости 3-я**) через Lens Selection Framework и зафиксировать *Paradigm → Theory → Construct → Model* + критерий интеграции. Детали: `ddx-lens-selection-framework.mdc`.

### Источник 3: от Рефлексии (Meta-Analysis)

Мета-позиция (Интенция 4.1, Объект C.2) -> "Моя эмоция -- диагностический сигнал?" -> гипотеза-кандидат.

### Принудительная генерация через Ось 2

Для каждого H1 или кластера данных последовательно применить:

- [ ] **1.0 Рефрейминг:** "Что если посмотреть иначе?" (смена позиции, темпорального контекста)
- [ ] **2.0 Деконструкция:** "Что если это устроено не так?" (экспликация допущений)
- [ ] **3.0 Трансформация:** "Что если это нечто совершенно иное?" (инверсия, семантическая трансформация)

**Правило полноты:** обязан применить эвристики из **каждой** мета-категории.

Детали эвристик: `ddx-generative-algorithm.mdc`. Состязательные эвристики (Режим 1): Skill `ddx-activate-mode`.

## 2. Workflow "Формализация H1/H0" (Formalization)

### Шаг 1: Формулирование H1

Превратить идею в **фальсифицируемое** утверждение. Критерии: конкретность, операционализируемость, ограниченность.

### Шаг 2: Операционализация (лестница абстракции)

| Этап | Вопрос | Результат |
|------|--------|-----------|
| Концептуализация | "Что именно мы имеем в виду?" | Определение |
| Дименсионализация | "Из каких компонентов?" | Список дименсий |
| Операционализация | "Как узнаем в тексте?" | Индикаторы для каждой дименсии |

Результат -> **Операциональный словарь проекта** (шаблон в [templates.md](templates.md)).

### Шаг 3: Формулирование H0

Через Ось 2 (см. раздел 1) -> наиболее сильная, правдоподобная альтернатива (не "соломенная").

### Шаг 4: Критерии фальсификации

Для **обеих** гипотез: "При каких наблюдениях гипотеза будет опровергнута?"

### Шаг 5: Фиксация

Обе гипотезы -> H-card (шаблон ниже) -> СМГ-реестр.

## 4. Workflow "Fast Falsification" (Сценарий 2.1.2)

### 5 шагов

1. **Мета-позиция:** сделать H1 явной и операциональной (лестница абстракции).
2. **Конструирование H0:** через Ось 2, самая сильная альтернатива.
3. **Поиск:** (a) данные ПРОТИВ H1 + (b) данные ЗА H0. Каждое свидетельство -> аннотация.
4. **Сравнение + Байесовское обновление:** оценить по объяснительной силе и экономности. Обновить уверенность (см. ниже).
   - **Если сравниваются конкурирующие гипотезы (особенно H1 vs H-Adv из Mode 1):** выполнить формальную сравнительную оценку через **Матрицу доказательств** (Прил.5, 1.2): строки `H-###`, столбцы `A-###`, значения `+/-/~ /0`. Детали: `ddx-smh-advanced.mdc`. Шаблон: [templates.md](templates.md).
5. **Решение:** H1 выдержала / H1 фальсифицирована / Неопределенность.

### При неопределенности

-> Эскалация: Skill `ddx-activate-mode` (Mode 1 для генерации третьей альтернативы; Mode 3 если H1 "слишком гладкая").

## 5. Байесовский принцип обновления (1.3.4)

### Трехшаговый цикл

1. **Prior:** начальная уверенность (Ось 5) при генерации.
2. **Evidence:** новое свидетельство оценивается по:
   - **Направление:** Подтверждает (+) / Опровергает (-) / Нерелевантно (0)
   - **Вес:** Сильное (маловероятно, если H неверна) / Слабое
3. **Posterior:** обновление уверенности.

| Свидетельство | Эффект |
|--------------|--------|
| Сильное (+) | Уверенность значительно повышается |
| Слабое (+) | Незначительно повышается |
| Сильное (-) | Резко падает (возможна фальсификация) |
| Слабое (-) | Незначительно снижается |

**Правило:** оценивать каждое свидетельство для **обеих** гипотез (H1 и H0). Идеальное "решающее" свидетельство: сильное (+) для одной, сильное (-) для другой.

### Фиксация

В H-card, Блок 2.4: `[Дата]: Уверенность повышена/понижена на основании A-[ID]. Обоснование: ...`

## 5.1 Ковариативный анализ (Manual: 1.3.3)

Ковариативный анализ — это формальная дисциплина для **доказательства связей и различий**, чтобы не оставаться на уровне интуиции.

### 5.1.1 Корреляционный анализ (Manual: 1.3.3.3)

Используй в **Фазе 2** (каузальные/структурные связи) и **Фазе 3** (интегративные карты/петли):

1. **Выбор пары элементов:** понятия/переменные/узлы модели (напр. H-001 и H-007 или “страх критики” и “прокрастинация”).
2. **Поиск совместной встречаемости:** собрать `A-###`, где оба индикатора проявлены в одном контексте.
3. **Анализ характера связи:** последовательность (что раньше?), условность (одно запускает другое?), интенсивность (вместе усиливаются?).
4. **Формулировка гипотезы о связи:** записать ребро в Hypothesis Map и указать основание `A-###`.

Правило: **стрелка/связь без основания `A-###` недопустима**.

### 5.1.2 Дискриминантный анализ (Manual: 1.3.3.3)

Ядро дифференциальной диагностики в **Фазе 2** для пар H1 vs H0/H-Adv:

1. **Выбор конкурирующей пары:** `H1` и сильная `H0`.
2. **Операционализация обеих гипотез (Manual: 1.1.4):** индикаторы/дименсии.
3. **Ключевой дифференцирующий признак:** 1 признак, который должен быть при одной гипотезе и отсутствовать при другой.
4. **Целенаправленный поиск:** искать **именно** этот признак в тексте.
5. **Вывод + обновление:** обновить Evidence Matrix, Bayes log и статус гипотез.

Усиление (Прил.3): “решающий факт / crucial discriminator” = формулировка прогностического “пари” и поиск факта, который одновременно подтверждает одну гипотезу и опровергает другую.

## 6. Workflow "Воронка Валидации" (Validation Funnel)

### Когда запускать

Пул гипотез > 7-10, особенно после Режима 1. Базовой проверки (Сценарий 2.1.2) недостаточно.

### Этап 1: Грубый отсев (Pruning)

Для каждой гипотезы -- 4 фильтра:

- [ ] Фальсифицируемость: можно ли представить опровергающее наблюдение?
- [ ] Правдоподобие: не противоречит фактам Риска 1?
- [ ] Релевантность: отвечает на диагностический вопрос?
- [ ] Добавленная ценность: оправдывает ли стоимость проверки?

### Этап 2: Приоритизация (Ranking)

AHP-сравнение оставшихся по 4 критериям: объяснительная сила, простота, прогностическая ценность, прагматическая полезность.
Если построена сеть гипотез — добавить **оценку центральности** (Degree/Betweenness) для коррекции ранга.
Результат: 2-3 финалиста.

### Этап 3: Решающая фальсификация

Финалисты проходят через Fast Falsification (раздел 4 выше) с продвинутыми техниками: Ключевой дифференцирующий признак (T1), Анализ негативного пространства (T3), Кросс-масштабная консистентность (T2).

Шаблон рабочего листа Воронки и детали техник -- `ddx-validation-funnel.mdc`.
Шаблоны (worksheet, AHP, centrality, timeline) — `templates.md`.
