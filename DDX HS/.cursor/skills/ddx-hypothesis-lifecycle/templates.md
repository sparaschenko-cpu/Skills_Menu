## Матрица доказательств (Evidence Matrix, Прил.5 / 1.2)

Используй для **формальной сравнительной оценки** конкурирующих гипотез (напр. H1 vs H0 или H1 vs H-Adv).

| Hypothesis \ Evidence | A-___ | A-___ | A-___ | A-___ |
|-----------------------|-------|-------|-------|-------|
| H-___ (H1)            |   +   |   -   |   0   |   ~   |
| H-___ (H0 / H-Adv)    |   -   |   +   |   0   |   ~   |

Легенда: `+` поддерживает, `-` фальсифицирует, `~` уточняет/модифицирует, `0` нейтрально.

## Качественный AHP (Qualitative AHP, Прил.3 / 2.2)

**Цель:** ранжировать гипотезы текстовой аргументацией (не числами).

### Матрица сравнений (пример для 3 гипотез)

| vs | H-001 | H-002 | H-003 |
|---|---|---|---|
| **H-001** | — |  |  |
| **H-002** |  | — |  |
| **H-003** |  |  | — |

### Как заполнять ячейку (H-x vs H-y)

Для каждой пары дать краткий вывод по 4 критериям:

- Объяснительная сила
- Простота (Оккам)
- Прогностическая ценность
- Прагматическая полезность

Формат ячейки:

`H-x > H-y по [критерий1, критерий2]; H-y > H-x по [критерий3]. Итог: H-x (слабое/среднее/сильное предпочтение).`

### Итог ранга

- Ранг = частота предпочтения по всем сравнениям (качественно).
- Зафиксировать 1–2 главные причины ранга (для прозрачности).

## Centrality worksheet (Анализ центральности, Прил.3 / 2.3)

Используй, если построена сеть гипотез (Прил.5 / Карта гипотез).

| H-ID | Degree (степень) | Betweenness (посредничество) | Роль | Как влияет на приоритет |
|---|---:|---:|---|---|
| H-___ | High/Med/Low | High/Med/Low | Hub/Bridge/Peripheral | Поднять/Оставить/Снизить |

Правило: гипотеза с высоким AHP-рангом **и** высокой центральностью получает максимальный приоритет проверки.
