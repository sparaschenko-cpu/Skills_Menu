### Агентный цикл (скелет)

```python
while True:
    response = await client.messages.create(
        model="claude-opus-4-6", max_tokens=8096,
        tools=tools, messages=messages
    )
    if response.stop_reason == "end_turn":
        return extract_text(response)
    elif response.stop_reason == "tool_use":
        results = await execute_tools(response.content)
        messages += [assistant_msg(response), user_msg(results)]
    elif response.stop_reason == "max_tokens":
        raise ContextOverflowError()
    elif response.stop_reason == "pause_turn":
        await wait_for_human_approval()
```

### Хуки (для данного проекта)

```json
{ "hooks": { "PostToolUse": [
  { "matcher": "[критичные операции]",
    "hooks": [{ "type": "command", "command": "python scripts/audit.py '${CLAUDE_TOOL_NAME}' '${CLAUDE_TOOL_INPUT}'" }] }
]}}
```

**Правило:** Хуки = детерминированные гарантии для бизнес-правил. Промпт-инструкции = вероятностное соответствие.

### Иерархия эскалации

```
1 → retry с exponential backoff (network, таймауты)
2 → альтернативный инструмент / упрощённая задача
3 → human-in-the-loop (business errors, permission errors)
4 → graceful degradation + аннотация пробелов покрытия
5 → hard failure + alert (критические системные ошибки)
```

### Human-in-the-loop точки

```

## Production Checklist

```
Domain 1 — Оркестрация:
□ stop_reason: end_turn, tool_use, max_tokens, pause_turn — все обработаны
□ Таймауты на все субагентные вызовы
□ allowedTools координатора включает "Task"
□ Хуки для аудита критичных операций

Domain 2 — Инструменты:
□ Каждый инструмент объясняет "КОГДА вызывать" в description
□ Все ошибки: is_error + error_category + is_retryable
□ .mcp.json использует ${ENV_VAR} для секретов
□ ≤ 5 инструментов на агента

Domain 3 — Конфигурация:
□ CLAUDE.md настроен минимум на уровне проекта
□ .claude/rules/ покрывают безопасность и соглашения
□ Slash-команды задокументированы

Domain 4 — Промпты:
□ Системный промпт: явные измеримые критерии успеха
□ Структурированный вывод через tool_choice + JSON schema
□ Validation-retry loop с MAX_RETRIES и конкретными ошибками
□ Nullable поля для необязательных данных

Domain 5 — Надёжность:
□ Иерархия эскалации (5 уровней) определена
□ Human-in-the-loop точки реализованы
□ Субагенты возвращают структурированный контекст ошибки
□ Мониторинг токенов (алерт при > 80%)
